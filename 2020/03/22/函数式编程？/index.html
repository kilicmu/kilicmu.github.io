函数式编程？

March 22, 2020 2492

我最先了解函数式编程是以前在油管上面看了一个小姐姐的分享视频：使用 JavaScript 学习函数式编程。

我承认我看这个视频的初衷真的是小姐姐很漂亮(｡・`ω´･)。但是看过以后也就基本忘了，只记得小姐姐分享的，推荐尽量去使用函数代替其他操作，同样的输入要产生同样的相同输出，不要产生副作用等等。

我知道作为一个在学界很受欢迎的编程范式，函数式编程的知识与智慧绝对不仅仅与这些，但是事实上我实在是精力有限，每天瞎忙，还没有钱。所以也没有取深入了解它。

不过在 19 年初期更新的 React 16.8 引入 hooks 概念，并在 19 年下半年愈加流行。我就知道，应该把这东西拿起来了解一下了，下面的内容与其所是博客，我认为它算是对《JS 函数式编程指南整理》的总结与理解。由于本人水平有限，理解可能存在问题，不喜勿喷。

函数式编程的核心就是将一系列操作分解为简单的函数，然后将一系列过程简化为函数嵌套

在 JS 中函数作为一等公民显然很适合来做函数式编程，在 JS 中函数可以如任何基本数据类型一样操作，这带来了相当优秀的灵活性。

总结就是使用函数式编程可以简化代码，降低耦合，避免 this 的使用。

性能问题

不得不说，函数式编程存在一些性能问题，可以参考：JavaScript 函数式编程存在性能问题么？

不过引入函数式编程本来也不是为了锱铢必较性能。而是要开发者把优化交给引擎，更多去保证代码的把可读性，可维护性，可测试性。

下面我们将打开函数式编程的大门。

几个概念

纯函数

特点：执行期间不依赖任何函数外部的数据或状态变化，必须仅仅依赖函数的输入决定输出。即相同输入对应相同输出，且不应去改变输入。

为什么要追求使用纯函数？

可移植性

JavaScript 的设定中，可移植性可以意味着把函数序列化（serializing）并通过 socket 发送。也可以意味着代码能够在 web workers 中运行。

纯函数与命令式编程不同，它与环境无关，只要我们愿意，可以在任何地方运行它。

自文档化

由于纯函数依赖单一，所以可以通过函数签名（函数名）来获取足够的信息。

可测试性

由于纯函数的特性可以使用一个最真实的断言来判断函数的正确性。

合理性

纯函数具有引用透明性的性质，如果一段代码可以替换成它执行所得的结果，而且是在不改变整个程序行为的前提下替换的，那么我们就说这段代码是引用透明的。

并行代码

纯函数也不会因副作用而进入竞争态，所以可以进行大量的并行操作而不用考虑资源问题。
