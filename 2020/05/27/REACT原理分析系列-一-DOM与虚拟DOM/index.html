虚拟DOM

什么是虚拟DOM

我的理解下，虚拟DOM就是使用JS的对象，抽象了DOM的具体实现。

为什么要有虚拟DOM

首先，我们必须明确DOM操作的性能是相对比较低的，但是作为前端开发者，不操作DOM显然不可能。在使用数据驱动开发应用，如果每次数据的变更都重新渲染所有DOM实在是过于损耗性能。

怎么解决？是的，只需要按需重绘就好了。

而virtual dom && diff为这种按需操作的实现提供了可能。如果你不知道diff算法可以看我以前的一篇文章（此处没有链接）。

因为这个系列是讲React的，所以此处使用React虚拟DOM的实现举例。
