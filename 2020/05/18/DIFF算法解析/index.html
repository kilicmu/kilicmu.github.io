React的diff算法

照例先挂文档：reconciliation

先挂个图示，从别人博客搬的：

简单说，React提出了一种复杂度为O(n)的算法取代传统的diff算法。

在React中，比较两棵树时，React 首先比较两棵树的根节点，有以下几种情况：

  • 不同类型元素

    销毁原有树建立新的树，对应的 DOM 节点也会被销毁，所有跟之前的树所关联的 state 也会被销毁。

  • 相同类型元素

    比对及更新有改变的属性，其中当更新 style 属性时，React 仅更新有所更变的属性

React会递归的去比对所有子节点。
